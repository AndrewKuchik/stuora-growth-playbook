# Архитектура сайта-навигатора «Stuora — Placement & Growth Playbook»

Документ для того, кто будет поддерживать страницу (и на будущее — если делать из этого продукт для других баров).

## 1. Зачем это
Одностраничный сайт, который отправляют ссылкой человеку (владельцу/маркетологу бара). Он открывает — и по блокам видит: где размещаться, что это даёт, сколько стоит, как подать заявку, и кнопка прямо на нужную страницу. Цель — чтобы получателю было **просто понять, что докрутить**, и **в один клик перейти к подаче**.

## 2. Аудитория и язык
- Получатель: владелец/маркетолог Stuora; вторично — партнёры (отели, тур-операторы).
- Язык: **английский** (по умолчанию). Архитектура готова к добавлению русского/латышского — см. §6.

## 3. Стек (максимально простой, бесплатный)
- **Один файл `index.html`** — HTML + встроенный CSS + ванильный JS. Ноль зависимостей, ноль сборки.
- Шрифты: Google Fonts (Fraunces, Instrument Sans, IBM Plex Mono) — подключаются ссылкой, на GitHub Pages грузятся без ограничений.
- **Хостинг: GitHub Pages** — бесплатно, ссылка вида `https://<user>.github.io/<repo>/`.
- Обновление: правим `index.html` → `git commit` → `git push` → сайт обновляется сам за ~1 минуту.

## 4. Структура страницы
1. **Masthead** — вордмарк, заголовок, подзаголовок, авто-считаемая статистика (сколько каналов упущено / бесплатных / приоритет 1).
2. **Toolbar (липкий)** — навигация по блокам + фильтры (All / Free / Paid / Priority 1) + переключатель темы + (скрытый) селектор языка.
3. **Блоки A–E + X** — каждый блок = сетка карточек-площадок.
4. **Footer** — как читать, оговорки, ключевые источники, отметка о мультиязычности.

Блоки: **A** — где нас нет + трафик · **B** — усилить существующее · **C** — партнёрства · **D** — премии · **E** — забрать чужие карточки (опционально) · **X** — что НЕ работает.

## 5. Как править контент
Весь контент — в массиве **`CH`** внутри `<script>` в `index.html`. Одна карточка = один объект:
```js
{ b:"a",              // id блока (a/b/c/d/e), или avoid:true для блока x
  cost:"free",        // free | paid | mixed  → бейдж + фильтр
  priority:1,         // 1 | 2 | 3            → цвет полоски слева
  status:"missing",   // missing | live | progress → бейдж статуса
  name:"Tripadvisor",
  domain:"tripadvisor.com/Owners",
  gives:{en:"…"},     // что даёт
  traffic:{en:"…"},   // охват/аудитория
  price:{en:"…"},     // цена
  steps:{en:["шаг 1","шаг 2"]},
  cta:{label:{en:"Open & claim"}, url:"https://…"}  // кнопка (или cta:null + note)
}
```
Добавить площадку = добавить объект в `CH`. Блоки описаны в массиве `BLOCKS`. Статистика в шапке пересчитывается сама.

## 6. Как добавить язык (ключевая возможность)
Контент data-driven, тексты обёрнуты в `{en:"…"}`, и функция `t()` берёт нужный язык.
Чтобы включить, например, русский:
1. В каждом переводимом поле добавить ключ: `gives:{en:"…", ru:"…"}` (и так для `traffic`, `price`, `steps`, `cta.label`, а также в `BLOCKS` для `title`/`sub`).
2. В массив `LOCALES` добавить `{code:"ru", label:"Русский"}`.
3. Всё. Селектор языка в тулбаре **появится автоматически** (он скрыт, пока язык один), переключение уже работает.

Латышский (`lv`) — так же.

## 7. Дизайн-система (токены)
- Тема задаётся CSS-переменными на `:root`; тёмная/светлая — через `prefers-color-scheme` и `data-theme` (переключатель + запоминание в `localStorage`).
- Палитра: латунь (`--brass`), бордо/oxblood (`--oxblood`), verdigris/патина для бейджа «Free», тёплые нейтрали (пергамент / near-black).
- Приоритет карточки кодируется полоской слева (1 — бордо, 2 — латунь, 3 — серый) — сканируется взглядом.
- Шрифты: Fraunces (заголовки), Instrument Sans (текст/интерфейс), IBM Plex Mono (данные/цены/домены).

## 8. Деплой и обновление
```bash
# первый раз — уже сделано скриптом публикации
git add -A && git commit -m "..." && git push
# GitHub Pages подхватит изменения автоматически
```
Ветка: `main`, папка: корень (`/`). Файл входа: `index.html`.

## 9. Идеи на будущее (если делать продукт)
- Вынести `CH`/`BLOCKS` в отдельный `data.js` или `content.json` — чтобы контент правился без кода.
- Шаблонизировать под любой бар/город (переменные: название, адрес, набор каналов).
- Добавить экспорт в PDF и трекер статуса («подано / одобрено»).
