# CLAUDE.md — правила работы в проекте «Stuora Growth»

## О проекте
Продвижение коктейль-бара **Stuora** (speakeasy, Рига, Audēju iela 15). В проекте:
- сайт-навигатор по каналам продвижения (публикуется на GitHub Pages);
- текстовый план-первоисточник.

## ГЛАВНОЕ — как работать эффективно (экономия кредитов)
1. Работай ТОЛЬКО по конкретному запросу и с конкретными файлами. Не читай проект целиком «на всякий случай».
2. Сначала загляни в этот файл и в «Карту файлов» ниже — не сканируй всё подряд.
3. Источник фактов — `Stuora-plan-prodvizheniya.md`. Бери оттуда, не перепроверяй заново без явной просьбы.
4. Не перезапускай тяжёлые многоагентные исследования, если пользователь явно не попросил.
5. Точечные правки: `Edit` по нужному месту, а не переписывание всего файла.
6. Для нового исследования запускай ОДНОГО профильного агента строго под тему, а не общий обход.
7. Не дублируй уже сделанную работу — сверься с этим файлом и памятью проекта.

## Карта файлов
- `index.html` — сам сайт. Весь контент площадок — в массиве **`CH`** внутри `<script>`; блоки — в **`BLOCKS`**. Правки контента = правки этих массивов.
- `ARCHITECTURE.md` — как устроен сайт, как править контент, как добавить язык.
- `Stuora-plan-prodvizheniya.md` — полный текстовый план (первоисточник контента).
- `README.md` — краткое описание + живая ссылка.

## Деплой (важно)
- Хостинг: **GitHub Pages через GitHub Actions** (`.github/workflows/deploy-pages.yml`) + файл `.nojekyll`.
- НЕ использовать legacy-Jekyll — он падал со сборкой (→ 404).
- Обновление: правишь файл → `git add -A && git commit -m "…" && git push` → сайт пересобирается сам за ~1–2 мин.
- Репозиторий (публичный): https://github.com/AndrewKuchik/stuora-growth-playbook
- Живой сайт: https://andrewkuchik.github.io/stuora-growth-playbook/

## Общение с пользователем
- По-русски, простыми словами (новичок в VS Code/git). Минимизировать ручные шаги — делать за него.
- Коммиты и бэкап на GitHub — сам, без напоминаний; описания коротко по-русски.
